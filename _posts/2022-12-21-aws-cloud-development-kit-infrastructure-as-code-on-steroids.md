Infrastructure as Code (IaC) ist weit verbreitet, aber viele Tools nutzen eigene deklarative Sprachen. Das AWS Cloud Development Kit (CDK) erlaubt hingegen, Infrastruktur mit bekannten Programmiersprachen wie TypeScript, Python oder Java zu modellieren, inklusive IDE-Support, Testintegration und einem reichen Ecosystem. Gleichzeitig kann eine zu hohe Flexibilität Code-Wartung und Übergaben erschweren.
